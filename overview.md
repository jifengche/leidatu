# 后台管理模块集成完成

## 功能说明
在主页面退出登录按钮旁新增「后台管理」入口，用于管理系统登录账号。

## 主要变更

### 新增文件
- `admin.html`：后台管理页面，统一头部风格，展示账号列表
- `css/admin.css`：后台管理页面专属样式，含表格、标签、弹窗
- `js/admin.js`：账号管理交互逻辑（新增、编辑、重置密码、停用/启用、删除）

### 修改文件
- `js/config.js`
  - 新增 `accounts` 和 `currentUser` 本地存储 key
  - 新增默认管理员账号列表配置
- `js/auth.js`
  - 账号从 localStorage 列表读取，支持多账号管理
  - 登录时校验账号存在性、密码、启用状态
  - 新增 `AccountManager` 对象，提供账号 CRUD、重置密码、切换状态等方法
  - 修复 `init()` 仅在登录页执行初始化逻辑，避免其他页面重定向
- `main.html`
  - 在退出登录左侧新增「后台管理」按钮
  - 引入 `js/auth.js` 以使用 `AuthHelper` / `AccountManager`
- `css/main.css`
  - 新增 `.btn-admin` 按钮样式（蓝色主调，hover 反白）
- `js/app.js`
  - 根据当前登录账号显示姓名
  - 仅管理员账号显示后台管理按钮并绑定跳转事件

## 权限规则
- 管理员账号：可进入后台管理页面，增删改查所有账号
- 教师账号：可登录使用系统主功能，但不显示后台管理入口
- 演示模式：不显示后台管理入口
- 安全限制：不能删除当前登录账号；不能删除最后一个管理员账号

## 测试验证
已通过浏览器自动化测试验证以下流程：
1. 使用默认 admin/123456 登录
2. 主页面正确显示「后台管理」按钮
3. 进入后台管理页面，列表展示默认管理员
4. 新增教师账号 → 编辑账号 → 重置密码 → 停用 → 启用 → 删除
5. 返回主页正常
